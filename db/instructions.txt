=========================================================
INSTRUCCIONES PARA LEVANTAR LA BASE DE DATOS CON DOCKER
=========================================================

Requisitos previos:
- Tener Docker Desktop instalado y en ejecución.
- Tener Docker Compose habilitado.
- Tener configurado el archivo `.env` dentro de esta carpeta.

---------------------------------------------------------
1. Abrir una terminal
---------------------------------------------------------

Puede ser CMD, PowerShell, Git Bash o la terminal de VS Code.

---------------------------------------------------------
2. Navegar a la carpeta de la base de datos
---------------------------------------------------------

cd db

---------------------------------------------------------
3. Levantar el contenedor de PostgreSQL
---------------------------------------------------------

docker compose up -d

Esto iniciará el contenedor en segundo plano.

---------------------------------------------------------
4. Verificar que el contenedor esté funcionando
---------------------------------------------------------

docker ps

Debería aparecer el contenedor:
rednorte-postgres-dev

---------------------------------------------------------
5. Ver logs en tiempo real (opcional)
---------------------------------------------------------

docker compose logs -f

Útil para verificar errores o el estado de inicialización
de la base de datos.

---------------------------------------------------------
6. Detener los contenedores
---------------------------------------------------------

docker compose stop

---------------------------------------------------------
7. Detener y eliminar los contenedores
---------------------------------------------------------

docker compose down

IMPORTANTE:
Este comando NO elimina los datos almacenados en el volumen
de Docker.

---------------------------------------------------------
INFORMACIÓN ADICIONAL
---------------------------------------------------------

- Puerto expuesto:
  5432 -> 5432 (o el puerto configurado en DB_PORT en el .env)

- Base de datos inicial:
  bd_rednorte

- Variables de entorno:
  configuradas en el archivo `.env`

- Persistencia de datos:
  volumen Docker `postgres_data`
